# Week 3 — Requirements Gathering  
## Software Requirements Specification (SRS)

---

## 1. Overview  

This document defines the core features, user requirements, and system functionality for the Adelaide Care Connect platform.  

The goal is to translate the identified business problem (Week 1) and market insights (Week 2) into clear system requirements.

---

## 2. Primary Users  

| User Type | Description |
|----------|------------|
| Participant | NDIS client who books and manages services |
| Staff | Support worker who manages assigned shifts |
| Administrator | Manages users, bookings, and system operations |

---

## 3. User Personas  

### Persona 1: Participant (NDIS Client)
- Needs simple and accessible system  
- Wants to book services easily  
- Wants to manage personal information and documents  

### Persona 2: Staff (Support Worker)
- Needs to view and manage shifts  
- Wants flexibility to accept/decline shifts  
- Needs clear information about participants  

### Persona 3: Administrator
- Needs full control over system operations  
- Manages participants, staff, and services  
- Requires reporting and monitoring tools  

---

## 4. Core Features  

### Participant Features
- Register and login  
- Book and manage services  
- Upload and view documents  
- Chat with AI chatbot  

### Staff Features
- View assigned shifts  
- Accept or decline shifts  
- Manage availability  

### Admin Features
- Manage participants and staff  
- Approve/reject bookings  
- Manage services and documents  
- View reports/dashboard  

---

## 5. User Stories  

### Participant User Stories
- As a participant, I want to register an account so that I can access services  
- As a participant, I want to book a service so that I can receive support  
- As a participant, I want to upload documents so that my records are stored securely  
- As a participant, I want to use a chatbot so that I can get quick answers  

### Staff User Stories
- As a staff member, I want to view my shifts so that I know my schedule  
- As a staff member, I want to accept or decline shifts so that I can manage my availability  

### Admin User Stories
- As an admin, I want to manage users so that the system stays organised  
- As an admin, I want to approve bookings so that services are properly assigned  
- As an admin, I want to view reports so that I can monitor system performance  

---

## 6. Functional Requirements (Summary)

- User registration and login  
- Role-based access control  
- Service booking system  
- Staff scheduling system  
- Document management  
- AI chatbot integration  
- Admin management dashboard  

---

## 7. Non-Functional Requirements (Summary)

- Secure data handling (encryption, authentication)  
- Fast performance (< 3 seconds load time)  
- Responsive design (mobile and desktop)  
- High availability (99% uptime)  
- Compliance with Australian Privacy Act  

---

## 8. Summary  

This SRS defines the system features, users, and requirements needed to develop the Adelaide Care Connect platform.  

It provides a clear foundation for system design and development in the following weeks.
