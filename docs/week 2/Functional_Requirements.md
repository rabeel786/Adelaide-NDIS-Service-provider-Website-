# Functional Requirements

The following functional requirements define the core features of the system.

| ID | Description | Priority | Module |
|----|------------|----------|--------|
| FR-01 | System allows participants to register with personal and NDIS details | High | Authentication |
| FR-02 | System validates NDIS number format (9-digit numeric) | High | Authentication |
| FR-03 | System authenticates users using email and password | High | Authentication |
| FR-04 | System implements role-based access (Participant, Staff, Admin) | High | Authentication |
| FR-05 | Participants can view and update their profile details | High | Participant |
| FR-06 | System displays available services by category and date | High | Booking |
| FR-07 | Participants can submit service booking requests | High | Booking |
| FR-08 | System sends booking confirmation via email | Medium | Booking |
| FR-09 | Participants can cancel bookings (with conditions) | High | Booking |
| FR-10 | Staff can view their assigned shifts | High | Staff |
| FR-11 | Staff can accept or decline shifts | High | Staff |
| FR-12 | Staff can request shift changes/swaps | Medium | Staff |
| FR-13 | Users can upload documents (PDF/DOCX) | High | Documents |
| FR-14 | Users can securely access their own documents | High | Documents |
| FR-15 | System integrates AI chatbot for user queries | High | Chatbot |
| FR-16 | Chatbot maintains session-based conversation history | Medium | Chatbot |
| FR-17 | Public users can submit contact enquiries | Medium | Public |
| FR-18 | System sends enquiry notifications to admin | Medium | Public |
| FR-19 | Admin can manage participants and staff (CRUD) | High | Admin |
| FR-20 | Admin dashboard displays system statistics and reports | High | Admin |

## Summary
These functional requirements ensure the system supports all key operations including user management, booking, scheduling, and communication.
